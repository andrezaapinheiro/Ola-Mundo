Olá, Mundo!

Neste curso foi apresentado as definições de Git e GitHub. Também foi possível criar branches e para compreender a dinâmica no trabalho em equipe. Todas as pessoas executando paralelamente sem interfeir no main, utilizando o processo de merge , unificando o histórico da programação ao final da sua construção. Ou seja, todo o gerenciamento do repositório acontece de forma paralela sem quebrar o código que está funcionando no branch master. A função merge, permite combinar duas ou mais branches inserindo todas as informações finais no main.

Git

é um sistema de controle de versão de códigos. Nele é possível, através do gerenciamento de ramificações, que várias pessoas de um time trabalhem em paralelo e integrem ao programa mantendo a ordem de todas as partes do programa. E também é um sistema que pode controlar as mudanças para evitar erros.

Github

é uma plataforma feita para hospedagem de códigos de programação. Os objetivos principais são o controle e a colaboração, permitindo que usuários desenvolvam projetos juntos, estando em qualquer lugar do mundo.
